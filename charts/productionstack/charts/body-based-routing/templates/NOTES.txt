Body-Based Router (BBR) installed as a cluster-wide workload singleton.

  Release          : {{ .Release.Name }}
  Namespace        : {{ include "bbr.namespace" . }}
  Service          : {{ .Values.bbr.name }}.{{ include "bbr.namespace" . }}.svc.cluster.local:{{ .Values.bbr.port }}
  Serving          : plaintext HTTP/2 (--secure-serving=false, no upstream TLS)

  This chart renders only the BBR Deployment + Service + cluster-scoped
  RBAC. It does NOT render an EnvoyFilter. The per-namespace gateway
  wiring (the ext_proc EnvoyFilter that injects BBR into a Gateway's
  HCM, scoped to that Gateway pod) is rendered by charts/modelharness.
  Make sure modelharness's `bbr.namespace` matches the namespace above
  so its EnvoyFilter targets this Service's cluster FQDN.
